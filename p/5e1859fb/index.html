VSCode 基本使用
2021-11-15|开发
字数总计: 1.3k|阅读时长: 5 分钟

VSCode，天下第一编辑器，古人云，欲善其事，必先利其器。

安装

官网下载，默认的安装方式即可，会注册为可编辑文件的默认编辑器以及加入环境变量(code命令)。

官方还提供一insider 版本，预览版，每天一更，也可以使用，这样就有两个vscode了。

官方也提供了一个便携版，绿色免安装，解压后，程序根路径添加 /data/tmp 两个文件夹，就可以作为一个独立的编辑器，升级需要自己将数据文件复制到新的解压的程序路径下就行。

插件

vscode第一次打开现在是默认会检测当前系统语言然后装相应的语言包插件，剩下的插件完全自己搭配

用户配置文件

settings.json

这个配置文件，要会改，其实和terminal调教差不多，而且官方都有数值参看的。

其他

Windows系统大陆用户会有 Ctrl + Space 被中文输入法切换占用的情况（情况少数），导致Ctrl空格唤不出来提示，需要去更改输入法热键，可以将Ctrl + Space 去掉，用Shift换中英文就够了

工作环境配置

  1. 支持以下预定义变量：
    ${workspaceRoot} -VSCode中打开文件夹的路径 the path of the folder opened in VS Code
    ${workspaceFolder} -在VS Code中打开的文件夹的路径
    ${workspaceFolderBasename}-在VS Code中打开的文件夹名称，不带任何斜杠（/）
    ${file} -当前打开的文件
    ${relativeFile} -相对于当前打开的文件workspaceFolder
    ${relativeFileDirname} -相对于当前打开的文件的目录名workspaceFolder
    ${fileBasename} -当前打开的文件的基本名称
    ${fileBasenameNoExtension}-当前打开的文件的基本名称，没有文件扩展名
    ${fileDirname}-当前打开的文件的目录名
    ${fileExtname} -当前打开的文件的扩展名
    ${cwd} -启动时任务运行器的当前工作目录
    ${lineNumber} -活动文件中当前选择的行号
    ${selectedText} -活动文件中的当前选定文本
    ${execPath}-正在运行的VS Code可执行文件的路径
    ${defaultBuildTask} -默认构建任务的名称
