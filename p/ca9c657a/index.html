Ubuntu 系统装配
2021-12-07|开发
字数总计: 2.1k|阅读时长: 7 分钟

Ubuntu, debian 好大儿，Linux 桌面端的佼佼者，稳定，用户友好。

前叙

本指南目的是让使用人员快速接纳 Linux，故选取 Ubuntu 作为参考，linux 换皮不换药，真正愿意使用 linux 的人员必然钟情于命令行操作，当然，便捷优雅的交互 UI 体验也是不可或缺的，这也正是 linux 生态难以养成的困难之一。

现在时间为 2021 年 9 月 17 日，各种软件如 QQ 等也有了良好能用的官方或三方产品，私以为，wayland 图形技术也大趋所向，现在投入 linux 的怀抱差不多是时候了。

面向版本：Ubuntu 20 +

个人建议版本：Ubuntu 20.04 LTS

安装

硬件参考

这是系统安装的硬件需求，不满足此需求的不是不能装，而是考虑兼容性、稳定性，以及综合体验给出的最低需求。

硬件 规格
CPU Intel Gen6+ / AMD Ryzen gen1+
内存 4GB +
存储 64GB+ SSD
显示器 1080P (1920x1080 像素)
无线网卡/蓝牙 可选

另外需 8GB+ 容量的 U 盘作为系统镜像安装盘。

安装模式

主板 BIOS 设置为 UEFI 引导模式（这是当前主流的系统引导方式），故而磁盘分区表也应格式化为 GPT 类型。

我们以一块 500GB 的固态硬盘为例，Bios 选择 U 盘启动：

警告：安装系统，不要联网，不要选择安装下载更新，尽可能选择最小化安装

  1. 安装过程中磁盘分配方式采用手动分区

建议分区如下

分区 分区文件系统格式 Linux 挂载点 分区说明 容量要求
sda1 fat16 /boot/efi 引导分区，主板过自检之后就会读取其中的引导文件，进而加载系统 100MB
sda2 ext4 /boot 启动分区，负责配置系统运行基本条件，加载内核等 500MB
sda3 ext4 / 挂载系统所有目录(排除已经自定义挂载到某分区的路径) 20G
sda4 ext4 /usr 用户通用程序和加载资源存储的位置 60G
sda5 ext4 /var 程序运行信息，执行单元加载的位置 40G
sda6 ext4 /opt 独立的三方用户程序的存储位置 20G
sda7 ext4 /home 用户私人家目录 剩余全部(记得留 Swap 区域)
sda8 swap 交换内存，内存剩余不足时，充当可读写的内存用 内存小于 8G 设置为等同于内存容量，大于 8G 则设置为 8G
  1. 安装过程中，会要求设置用户正式姓名，用户昵称(将作为识别的用户名 user)，主机名(hostname)，以及用户密码(password)。
  2. 时间区域选择 Asia/shanghai，linux 默认将主板上的硬件时间当做 UTC(世界标准时间)，而我们是 UTC+8，联网同步时间会将主板上的时间回调 8 个小时。
  3. 安装完成，退出安装系统，拔掉 U 盘，进 Bios 更改系统为已经安装的 Ubuntu

APT 包管理常用命令

  • 更新本地仓库记录 sudo apt update

  • 更新系统内核及程序 sudo apt upgrade 或者 sudo apt full-upgrade

  • 安装程序 sudo apt install <app_name>，可加多个程序名

  • 修复依赖安装 -f 修饰 install, 默认确认安装 -y 修饰 install

  • 移除应用sudo apt remove <app_name>

  • 移除应用并删除其配置 sudo apt purge <app_name>

  • 移除无用依赖sudo apt autoremove

  • 移除残留配置sudo apt autoclean

  • 搜索包apt search app_name

  • 查看包 apt show app_name

dpkg 包管理

  • 浏览包 dpkg --list 或 dpkg -l 搭配 grep 用来关键词查找很方便
